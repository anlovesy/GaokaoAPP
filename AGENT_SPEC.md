# 高考志愿智能体设计说明

## 智能体目标

服务对象是高考出分后的学生与家长，帮助他们在有限时间内完成更有逻辑的志愿筛选，而不是只看学校名气或只看分数。

智能体的核心任务：

1. 采集学生画像
2. 识别兴趣和职业方向
3. 将“学校、城市、专业、风险”做综合平衡
4. 生成冲 / 稳 / 保志愿建议
5. 对每个推荐给出清楚解释

## 当前原型已实现的能力

- 支持输入省份、分数、位次、兴趣爱好、职业规划、风险偏好和城市意向
- 根据关键词和兴趣标签匹配专业方向
- 根据估算位次阈值生成冲 / 稳 / 保院校专业推荐
- 输出推荐理由、填报策略和提醒信息

## 推荐逻辑

### 第一步：构建学生画像

输入信息包括：

- 省份
- 分数
- 全省位次
- 风险偏好
- 兴趣标签
- 职业规划文本
- 城市偏好

### 第二步：识别专业方向

系统根据学生的兴趣标签和职业规划文本，匹配专业方向，例如：

- 技术 / 编程 -> 计算机、软件工程、人工智能
- 医学 / 健康 -> 临床医学、口腔医学、药学
- 金融 / 商业 -> 金融学、会计学
- 法律 / 公共治理 -> 法学、行政管理

### 第三步：形成院校专业组合

每个院校定义：

- 城市
- 办学定位
- 适配标签
- 基础位次门槛
- 对应专业方向

系统通过“院校基础位次 + 专业偏移量”形成专业推荐门槛，再结合不同省份难度系数估算匹配区间。

### 第四步：按风险偏好分层

- 冲：有挑战，但学校层次更高
- 稳：录取概率与匹配度平衡
- 保：优先保障录取结果

## 下一步升级建议

如果要把当前原型升级成真正可用的志愿填报智能体，建议按这个顺序扩展：

1. 接入真实数据
2. 接入大模型问答
3. 加入筛选约束
4. 输出正式志愿表

### 1. 接入真实数据

优先补充这些数据源：

- 各省一分一段表
- 各院校近 3 年最低录取位次
- 各专业近 3 年最低录取位次
- 选科要求
- 招生计划数
- 学费和培养校区

### 2. 接入大模型问答

大模型可以负责：

- 将学生自由输入总结成结构化画像
- 解释某个专业适不适合该学生
- 回答“学校优先还是专业优先”
- 识别家长偏好与学生偏好的冲突
- 给出多种填报策略版本

### 3. 加入筛选约束

建议加入这些筛选项：

- 是否接受调剂
- 是否优先 985 / 211 / 双一流
- 是否必须在省会或一线城市
- 是否看重读研率
- 是否看重就业稳定
- 是否限制学费
- 是否限制专业冷热度

### 4. 输出正式志愿表

最终可以输出：

- 第一版冲稳保正式志愿清单
- 备选志愿清单
- 志愿顺序调整建议
- 不建议填报的高风险选项

## 建议的智能体提示词方向

如果后续接入大模型 API，可以把模型角色定义为：

“你是一名高考志愿规划顾问，需要根据学生的省份、位次、兴趣、职业规划和风险偏好，推荐大学与专业组合。你要优先考虑录取可行性、专业适配度和长期职业路径，并清楚解释推荐原因与风险。”

## 当前限制

当前版本仍然是演示原型，不具备以下能力：

- 不读取实时招生计划
- 不校验真实选科限制
- 不保证真实录取结果
- 不做最新年度官方数据比对

因此它更适合：

- 产品演示
- 原型展示
- 逻辑验证
- 后续二次开发底座
