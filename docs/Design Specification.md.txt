# AI 高考志愿顾问（Gaokao AI Advisor）

# UI / UX Design Specification v2.0

## 适用于 Codex 全站重构

---

# 一、项目目标（Mission）

本项目不是一个普通的高考志愿填报网站。

它是：

> 一个高端 AI 高考志愿顾问（AI Admission Strategist）。

它应该给用户的第一感觉是：

* 专业
* 高级
* 有科技感
* 有品牌感
* 值得信任

而不是：

* 教育网站
* 管理后台
* SaaS Dashboard
* 数据管理系统

整个网站应该更像：

Apple Product Page

Linear

Vercel

Framer

Raycast

ChatGPT Desktop

Federico Pian Portfolio

这是整个项目唯一需要遵循的设计方向。

# 三、总体设计原则

整个网站遵循：

"Experience First"

而不是：

"Function First"

用户进入网站以后，

不要看到：

平台介绍

功能介绍

AI 能力

工作流程

模型说明

系统说明

游客说明

正式用户说明

不要解释产品。

让用户直接体验产品。

遵循：

Apple 官网设计哲学：

"Less Interface, More Experience."

# 六、页面内容规范

删除所有介绍性内容。

包括：

平台介绍

平台结构

模型能力

当前数据状态

高校留影墙介绍

正式用户可以获得

游客体验边界

为什么这样设计

AI介绍

FAQ

帮助

教程

说明

提示框

所有长文本。

每个页面：

最多：

一个标题

一句副标题（20字以内）

一个主要按钮

即可。

# 七、首页（Landing Page）

首页采用 Scroll Storytelling。

Section 1：

Hero。

左侧：

超大标题。

一句价值主张。

CTA。

右侧：

全国高校图片自由漂浮。

鼠标视差。

Hover Motion。

Section 2：

高校图片墙。

自动滚动。

Hover 放大。

不要介绍。

Section 3：

真实志愿案例。

采用 Story Card。

Section 4：

开始体验。

结束。

首页不要：

Dashboard

统计数字

模型说明

系统介绍

数据卡片。

# 十六、最终目标

最终效果应该达到：

"不像一个教育网站。"

而像：

"一个价值数万元 AI 高考志愿咨询服务的数字产品。"

整个网站应具备：

品牌感

高级感

沉浸感

电影级滚动体验

优秀的空间留白

流畅的交互动画

而不是一个功能展示页面。

不要在现有 UI 上继续修修补补。

请将当前页面视为低保真原型（Wireframe）。

保留所有业务逻辑，从零重新设计整个视觉体系。
